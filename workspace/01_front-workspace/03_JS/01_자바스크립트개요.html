자바스크립트 개요

스크입트 언어란?

프로그램의 동작을 사용자 요구에 맞게 수행하도록 해주는 용도
빠르게 배울수 있고 짧은 소스 코드로 상호작용 되도록 고안

자바스크립트

웹 브라우저에서 많이 사용되는 인터프리터 방식의 스크립트 언어
=> 인터프리터 방식: 코드를 한줄씩 읽어가면서 바로 실행하는 방식
--> 코드에 문법 오류가 있으면 실행 시점에 알 수 있음

장단점

*장점

  • 컴파일 과정 없이 인터프리터로 코드를 한줄씩
    읽어가면서 바로 실행되기 때문에 수행속도 빠르다
  • 코드 작성이 간단해서 처음 접하기 수월함(단순한 구조/원칙/규칙) ex) 접근제한자 개념 없음, 변수 선언 시 자료형을 정해줄 필요없음

*단점

  • 웹 특화된 기술 -> 내부에서 제공하는 기술 제한적
  • html 문서 내에 작성하면 외부에 공개됨 => 보안 취약함

자바스크립트 사용법

  1. inline(인라인) : 태그 내에 간단한 소스코드를 작성헤서 사용하는 방법
  2. >internal(내부) : html 문서 내에 소스코드를 작성헤서 사용하는 방법(script 태그)
  3. >external(외부) : 별도의 자바스크입트파일(.js)로 소스코드를 작성헤서 현재 문서에 연결해서 사용하는 방법 (script 태그 - src 속성 사용해)

인라인 방식

[표현법] <태그명 on이벤트명(onClick)="자바스크립트 코드 작성(해당 요소에 이벤트가 발생되었을때 실행할 내용)">




*내부 방식

script 태그 영역 안에 소스코드 작성
=> head 태그, body 태그 내에 모두 작성 가능!



*외부 방식

외부 파일(.js)을 작성하여 현재 문서에 연결
=> script 태그를 사용해 src 속성에 파일 경로 작성
